Page Replacement Algorithms

Let's understand and visualise 3 categories of Page Replacement Algorithms.

What is Page Replacement?

In an operating system that uses paging for memory management, a page replacement algorithm is needed to decide which page needs to be replaced when new page comes in.

Important Terminologies

  • Page Fault:A page fault is an interruption that occurs when a software program attempts to access a memory block not currently stored in the system's RAM.The page fault is also called page miss.
  • Page Hit:When we want to load the page on the memory, and the page is already available on memory, then it is called page hit.
  • Page Fault Rate: A page fault will occur for every certain amount of main memory access,which is called page fault rate.
  • Page Hit Rate: The percentage of times that a particular page number is found in the TLB is called the Page hit ratio.
